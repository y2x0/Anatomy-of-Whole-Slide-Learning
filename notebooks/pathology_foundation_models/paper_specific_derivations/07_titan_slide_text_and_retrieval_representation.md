# TITAN: Slide-Text And Retrieval Representation

References: [TITAN paper](https://www.nature.com/articles/s41591-025-03982-3),
[official implementation](https://github.com/mahmoodlab/TITAN).

TITAN is not one image-text contrastive loss applied to a bag of patches. It is
a three-stage slide model with two spatial resolutions for language alignment:
vision-only feature-grid pretraining, ROI-caption alignment, and WSI-report
alignment.

## 1. Input Feature Grid

TITAN uses CONCHv1.5 to encode non-overlapping `512` by `512` tissue patches at
20x magnification. Each patch feature has dimension `768`:

```math
h_{ab}
=
f_{\mathrm{CONCHv1.5}}
\left(
x_{ab}^{512}
\right)
\in
\mathbb{R}^{768}.
```

The features are placed back into the slide grid at coordinates `(a,b)`:

```math
H
=
\left[
h_{ab}
\right]_{(a,b)\in\Omega_i}.
```

This patch encoder acts as TITAN's embedding layer. The slide transformer does
not receive raw pixels and cannot recover information removed by CONCHv1.5.

## 2. Stage 1: Vision-Only iBOT

For each WSI, TITAN samples a `16` by `16` feature-grid region corresponding to
an `8192` by `8192` pixel field of view. From this region it samples two global
`14` by `14` crops and ten local `6` by `6` crops. If `V` is the set of feature
grid views, iBOT gives:

```math
\mathcal L_{\mathrm{Stage1}}
=
\sum_{v_t\in V_{\mathrm{global}}}
\sum_{v_s\in V:
v_s\neq v_t}
H
\left(
p_t(v_t),p_s(v_s)
\right)
+
\lambda_{\mathrm{MIM}}
\mathcal L_{\mathrm{masked\mbox{-}token}}.
```

The slide encoder is a compact ViT with six transformer layers, twelve heads
of dimension `64`, model width `768`, and hidden width `3072`. Its patch
embedding layer is replaced by an MLP over the `768`-dimensional CONCH
features. Stage 1 produces the vision-only model `TITAN_V`.

## 3. Two-Dimensional ALiBi

Absolute position embeddings extrapolate poorly when the training crop is
shorter than a full WSI. TITAN adds an attention bias based on Euclidean grid
distance. For query `q_i` and key `k_j` at feature-grid locations
`r_i=(i_x,i_y)` and `r_j=(j_x,j_y)`, head `h` uses:

```math
b_{ij}^{(h)}
=
-m_h
\left|
r_i-r_j
\right|_2
=
-m_h
\sqrt{
\left(i_x-j_x\right)^2
+
\left(i_y-j_y\right)^2
}.
```

The attention weights are:

```math
A_{ij}^{(h)}
=
\mathrm{softmax}_j
\left(
\frac{q_i^{\top}k_j}{\sqrt d}
+
b_{ij}^{(h)}
\right).
```

The bias penalizes distant pairs but does not make attention local. TITAN trains
on `16` by `16` feature regions and extrapolates to whole-slide grids at test
time. This is a train-short/test-long contract, not proof of scale invariance.

## 4. Stage 2: ROI-Caption Alignment

Stage 2 aligns `423,122` `8192` by `8192` ROIs with fine-grained synthetic
captions generated by PathChat. Let the contextualized feature grid be `H_i`
and caption embedding be `t_i`. TITAN uses CoCa, whose contrastive component
can be written:

```math
s_{ir}
=
\frac{
z_i^{\top}t_r
}{
\left\lVert z_i\right\rVert_2
\left\lVert t_r\right\rVert_2
},
```

```math
\mathcal L_{\mathrm{ROI\mbox{-}CLIP}}
=
\frac{1}{2}
\left(
\mathcal L_{I\to T}
+
\mathcal L_{T\to I}
\right),
```

where:

```math
\mathcal L_{I\to T}
=
-\frac{1}{B}
\sum_{i=1}^{B}
\log
\frac{\exp(s_{ii}/\tau)}
{\sum_{r=1}^{B}\exp(s_{ir}/\tau)}.
```

CoCa adds an autoregressive captioning loss. If `w_{i,1:L}` is the target
caption and `p_\omega` is the multimodal decoder:

```math
\mathcal L_{\mathrm{cap}}
=
-\frac{1}{B}
\sum_{i=1}^{B}
\sum_{\ell=1}^{L}
\log
p_\omega
\left(
w_{i,\ell}
\mid
w_{i,<\ell},H_i
\right).
```

The Stage 2 representation is thus trained to be both retrievable against
captions and generative through a decoder.

## 5. Stage 3: WSI-Report Alignment

Stage 3 continues training on `182,862` WSI-report pairs. The input is enlarged
to a random `64` by `64` feature grid, corresponding to `32768` by `32768`
pixels, before whole-slide inference. The report stage uses the same CoCa
structure but a different context distribution:

```math
\mathcal L_{\mathrm{Stage3}}
=
\lambda_{\mathrm{ctr}}
\mathcal L_{\mathrm{WSI\mbox{-}report\mbox{-}CLIP}}
+
\lambda_{\mathrm{cap}}
\mathcal L_{\mathrm{report}}.
```

Stage 2 supplies detailed local morphology; Stage 3 supplies coarse diagnostic
and clinical language. The final TITAN representation is therefore shaped by a
curriculum over context scale, not merely by adding more caption pairs.

## 6. One Query Versus 128 Queries

TITAN uses two attentional poolers on the contextualized feature grid. A single
contrastive query produces the global representation used for retrieval and
zero-shot evaluation:

```math
z_{\mathrm{slide}}
=
\mathrm{Pool}_{1}(H_i).
```

The multimodal decoder receives `128` reconstruction-query tokens:

```math
R_i
=
\mathrm{Pool}_{128}(H_i)
\in
\mathbb{R}^{128\times768}.
```

These are different readout objects. The one-query embedding is not the same
statistic as the 128-token sequence used for report generation.

For a generic attentional pooler with query `q`:

```math
a_j
=
q^{\top}W_k h_j,
\qquad
\alpha_j
=
\frac{\exp(a_j)}{\sum_r\exp(a_r)},
\qquad
z_q
=
\sum_j\alpha_j W_vh_j.
```

Changing the number of queries changes the rank and capacity of the surviving
slide statistic.

## 7. Retrieval Geometry

For slide embeddings `z_i` and report embeddings `t_r`, retrieval uses cosine
similarity:

```math
s(i,r)
=
\frac{z_i^{\top}t_r}
{\left\lVert z_i\right\rVert_2
 \left\lVert t_r\right\rVert_2}.
```

The top-`K` report set is:

```math
\mathcal N_K(i)
=
\underset{A\subseteq[R],\,|A|=K}{\arg\max}
\sum_{r\in A}s(i,r).
```

This is a memory-dependent readout. Changing the archive, duplicate policy,
patient exclusion rule, or text normalization changes retrieval results even
when the slide encoder is frozen.

## 8. What TITAN Preserves

The pretraining contract can preserve:

```text
local morphology through CONCHv1.5 features;
within-ROI spatial context through Stage 1 iBOT;
fine-grained morphology-language alignment through Stage 2;
slide-level diagnostic/report alignment through Stage 3;
retrieval geometry through the one-query slide embedding;
report-generation state through the 128-query decoder tokens.
```

It does not guarantee causal diagnosis. A report can contain patient-level
labels, templated phrases, or an institution-specific writing style. Synthetic
captions can also amplify the visual-language model's own shortcuts.

## C/R/G/S Placement

```text
C:
    ViT over 2D feature grids with distance-biased attention and train-short,
    test-long extrapolation

R:
    one-query slide embedding for retrieval; 128-query token set for generation

G:
    CONCHv1.5 feature geometry, 2D Euclidean ALiBi, and shared slide-text space

S:
    iBOT teacher targets, 423K PathChat ROI captions, 183K pathology reports,
    and autoregressive CoCa text targets
```

## Failure Matrix

| Failure | Mathematical source | Consequence |
|---|---|---|
| frozen patch bottleneck | `h_j=f_CONCH(x_j)` is non-injective | lost tile detail cannot be recovered |
| crop-to-slide gap | Stage 1 sees `16` by `16` grids | full-slide interactions are extrapolated |
| ALiBi geometry error | distance bias is only an inductive prior | curved tissue and irregular anatomy can be misrepresented |
| caption shortcut | synthetic text is generated from visual-language priors | model can align to generated errors |
| report shortcut | reports contain labels or institution style | retrieval may reflect text metadata rather than morphology |
| readout mismatch | `Pool_1(H)` differs from `Pool_128(H)` | retrieval and generation explain different objects |
| archive dependence | neighbors depend on memory set | retrieval claims are not encoder-only claims |
