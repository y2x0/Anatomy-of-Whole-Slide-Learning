# MoCo Dictionary, Queue, And Staleness

MoCo changes the candidate mechanism rather than the InfoNCE formula. A queue
increases dictionary size, while a momentum encoder controls inconsistency
between keys produced at different optimization times.

Primary source:

- He et al. "Momentum Contrast for Unsupervised Visual Representation
  Learning." CVPR 2020. https://arxiv.org/abs/1911.05722

## Query And Key Encoders

At optimization step `t`, let:

```math
q_t
=
f_{\theta_{q,t}}
\left(
V^{(q)}
\right),
```

```math
k_t^{+}
=
f_{\theta_{k,t}}
\left(
V^{(k)}
\right).
```

The key branch is detached from gradient backpropagation. Only query parameters
receive the direct InfoNCE gradient.

## Queue Dictionary

Let the queue used to score the current query contain `K` normalized key
vectors:

```math
\mathcal{Q}_t
=
\left\{
k_{t-a_r}^{(r)}
\right\}_{r=1}^{K},
```

where `a_r` is the age of key `r` in optimization steps and is at least one for
the negative queue used before the current minibatch is enqueued.

For one query, MoCo uses:

```math
\mathcal{L}_{q,t}
=
-
\log
\frac{
\exp(q_t^{\top}k_t^{+}/\tau)
}{
\exp(q_t^{\top}k_t^{+}/\tau)
+
\sum_{r=1}^{K}
\exp(q_t^{\top}k_{t-a_r}^{(r)}/\tau)
}.
```

The queue decouples candidate count from current minibatch size.

## Momentum Key Update

MoCo updates the key encoder by:

```math
\theta_{k,t}
=
m\theta_{k,t-1}
+
(1-m)\theta_{q,t},
```

where:

```math
0\le m<1.
```

Unrolling `a` steps gives:

```math
\theta_{k,t}
=
m^a\theta_{k,t-a}
+
(1-m)
\sum_{r=0}^{a-1}
m^r\theta_{q,t-r}.
```

The current key encoder is an exponentially weighted history of query
encoders, plus a decaying contribution from its earlier state.

## Key Age And Parameter Mismatch

A queued key of age `a` was produced by:

```math
k_{t-a}
=
f_{\theta_{k,t-a}}(v).
```

The current query scores it using:

```math
q_t^{\top}k_{t-a}.
```

This compares vectors generated by different parameter states. Define encoder
drift:

```math
\Delta_{t,a}
=
\lVert
\theta_{k,t}-\theta_{k,t-a}
\rVert_2.
```

If the encoder is Lipschitz in parameters with constant `L_theta` at input `v`:

```math
\lVert
f_{\theta_{k,t}}(v)
-
f_{\theta_{k,t-a}}(v)
\rVert_2
\le
L_{\theta}(v)
\Delta_{t,a}.
```

Momentum reduces parameter drift but does not make old keys identical to
current-encoder keys. If the query is normalized, the induced similarity error
is bounded by:

```math
\left|
q_t^\top k_t
-
q_t^\top k_{t-a}
\right|
\le
\left\|
k_t-k_{t-a}
\right\|_2.
```

The corresponding contrastive-logit error is at most:

```math
\left|
\frac{q_t^\top k_t}{\tau}
-
\frac{q_t^\top k_{t-a}}{\tau}
\right|
\le
\frac{L_{\theta}(v)\Delta_{t,a}}{\tau}.
```

Queue staleness is therefore more consequential at low temperature, even when
the underlying encoder drift is unchanged.

## Queue Age Distribution

Suppose each step enqueues `B` keys after evaluating the current loss and the
queue length is `K`, with `K` divisible by `B`. Then the negative queue contains
approximately:

```math
R
=
\frac{K}{B}
```

minibatches of keys.

At the moment the current loss is evaluated, ignoring within-step ordering,
negative-key ages range over:

```math
1,2,\ldots,R.
```

The average age is:

```math
\overline a
=
\frac{R+1}{2}.
```

Increasing queue size at fixed enqueue batch size increases both dictionary
coverage and average staleness.

## Candidate Proposal Is Time-Mixed

The queue does not contain samples from one current embedding distribution. It
contains a mixture:

```math
\nu_t(k)
\approx
\frac{1}{R}
\sum_{a=1}^{R}
\left(
f_{\theta_{k,t-a}}
\right)_{\#}
p_V,
```

where the pushforward notation means the view distribution is encoded by
different historical key networks.

The density-ratio target is therefore time-dependent unless key encoders are
sufficiently consistent.

## Stop-Gradient Boundary

For queued and current keys:

```math
\frac{\partial\mathcal{L}_{q,t}}
{\partial\theta_{k,t}}
=
0
```

through backpropagation. The key encoder changes only through the momentum
update.

This creates two coupled dynamics:

```text
query dynamics:
    gradient descent on contrastive loss

key dynamics:
    exponential moving average of query parameters
```

They are not ordinary joint gradient descent on query and key parameters.

## Size-Consistency Tradeoff

Larger dictionary:

```text
more candidate coverage
higher possible identification difficulty
more old keys
greater memory and scoring cost
```

Larger momentum:

```text
slower key drift
more consistent queued representations
slower adaptation to the current query encoder
```

The two hyperparameters jointly define the candidate process.

## C/R/G/S Placement

```text
\mathcal{G}:
    queue support, key ages, and time-mixed candidate distribution

\mathcal{C}:
    query encoder and momentum key encoder

\mathcal{R}:
    normalized query/key projections

\mathcal{S}:
    positive key at index zero and queued negatives
```

## Dense Summary

MoCo solves one scaling problem by introducing one statistical approximation:

```math
\text{large dictionary}
\quad\Longleftrightarrow\quad
\text{historical candidate embeddings}.
```

Momentum controls, but does not eliminate, that historical mismatch.
