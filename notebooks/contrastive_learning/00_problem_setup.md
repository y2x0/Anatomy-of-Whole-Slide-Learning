# Contrastive Learning Problem Setup

Contrastive learning does not begin with a loss. It begins with a statistical
experiment that declares which observations should share information and which
observations should be distinguishable.

Primary anchors for this setup:

- van den Oord, Li, and Vinyals. "Representation Learning with Contrastive
  Predictive Coding." 2018. https://arxiv.org/abs/1807.03748
- Chen et al. "A Simple Framework for Contrastive Learning of Visual
  Representations." ICML 2020. https://arxiv.org/abs/2002.05709

## Source Object And Views

Let the latent source object be:

```math
X
\sim
p_X.
```

Two stochastic view kernels generate observations:

```math
V^{(1)}
\sim
\mathcal{A}_1(\cdot\mid X),
\qquad
V^{(2)}
\sim
\mathcal{A}_2(\cdot\mid X).
```

The positive-pair joint distribution is:

```math
p_{+}(v_1,v_2)
=
\int
p_X(x)
\mathcal{A}_1(v_1\mid x)
\mathcal{A}_2(v_2\mid x)
\,dx.
```

The product of view marginals is:

```math
p_{\mathrm{prod}}(v_1,v_2)
=
p_{V^{(1)}}(v_1)
p_{V^{(2)}}(v_2).
```

Contrastive learning distinguishes samples from `p_+` against candidates drawn
from a proposal intended to approximate one of these marginals.

## Positive Does Not Mean Same Class

The event:

```math
(V^{(1)},V^{(2)})
\sim
p_{+}
```

means that the pair was generated by the chosen positive mechanism. It does not
logically imply:

```math
Y(V^{(1)})
=
Y(V^{(2)}),
```

nor:

```math
U(V^{(1)})
=
U(V^{(2)}),
```

for an unobserved biological state `U`. Those equalities are assumptions about
the augmentation or pairing channel.

In pathology, a crop, stain transform, same-slide relation, same-patient
relation, cluster assignment, or report match defines a different `p_+`.

## Encoder And Projection Spaces

An encoder produces the downstream representation:

```math
h
=
f_{\theta}(v)
\in
\mathbb{R}^{d_h}.
```

A projection head produces the vector on which the pretraining objective acts:

```math
z
=
g_{\phi}(h)
\in
\mathbb{R}^{d_z}.
```

For normalized contrastive objectives:

```math
u
=
\frac{z}{\lVert z\rVert_2}
\in
\mathbb{S}^{d_z-1}.
```

These are three different objects:

```text
h:
    representation retained for downstream use

z:
    projection optimized by the pretraining objective

u:
    normalized point whose angles enter the contrastive logits
```

SimCLR explicitly discards the projection head after pretraining. Therefore the
geometry directly constrained by the loss is not automatically identical to
the retained representation geometry.

## Candidate Identification Experiment

Let `A` denote an anchor and let:

```math
I
\sim
\mathrm{Uniform}
\{1,\ldots,N\}
```

select the positive location in a candidate set. Generate:

```math
K_I
\sim
p_{K\mid A}(\cdot\mid A),
```

and for every other index:

```math
K_j
\sim
\nu_K,
\qquad
j\ne I.
```

The contrastive task is to infer `I` from:

```math
\mathcal{D}_N
=
\left(
A,K_1,\ldots,K_N
\right).
```

The proposal `nu_K` is part of the learning problem. In-batch negatives, a
memory queue, same-organ negatives, and cross-cohort negatives generally define
different proposals.

## Score And Softmax Classifier

Let a critic produce:

```math
s_{\psi}(A,K_j)
\in
\mathbb{R}.
```

The candidate-index classifier is:

```math
q_{\psi}
\left(
I=j
\mid
\mathcal{D}_N
\right)
=
\frac{
\exp(s_{\psi}(A,K_j))
}{
\sum_{r=1}^{N}
\exp(s_{\psi}(A,K_r))
}.
```

InfoNCE is the cross-entropy for this generated index label:

```math
\mathcal{L}_{N}
=
\mathbb{E}
\left[
-\log
q_{\psi}
\left(
I
\mid
\mathcal{D}_N
\right)
\right].
```

The supervised target is not an external class label. It is the positive
location generated by the contrastive experiment.

## Three Sources Of Geometry

Input relation geometry:

```math
p_{+},
\qquad
\nu_K.
```

Critic geometry:

```math
s_{\psi}(a,k).
```

Representation geometry:

```math
u_a^{\top}u_k,
\qquad
\lVert u_a-u_k\rVert_2.
```

These should not be conflated. Sampling declares which comparisons occur; the
critic scores them; optimization changes the embedding geometry.

## Contrastive And Negative-Free Families

Explicit-negative methods use a candidate denominator:

```math
\sum_{j=1}^{N}
\exp(s_{\psi}(A,K_j)).
```

Negative-free joint-embedding methods replace that denominator with other
anti-collapse mechanisms:

```text
BYOL and SimSiam:
    asymmetric prediction and stop-gradient targets

Barlow Twins:
    cross-correlation identity constraint

VICReg:
    invariance, variance, and covariance penalties
```

They still learn similarity, but not by candidate-index classification.

## C/R/G/S Placement

```text
\mathcal{G}:
    augmentation, pairing, candidate, and similarity geometry

\mathcal{C}:
    encoder or contextual encoder producing object representations

\mathcal{R}:
    pooling, projection, and normalization defining the compared object

\mathcal{S}:
    positive index, class-positive set, teacher target, or batch-moment target
```

## Core Audit Questions

```text
1. What random mechanism generates a positive?
2. From which distribution are comparison candidates drawn?
3. Which representation is scored and which one survives downstream?
4. Is similarity angular, Euclidean, bilinear, cross-modal, or graph-based?
5. What prevents collapse?
6. Which population quantity is estimated under the actual sampler?
7. Which latent distinctions are erased by the chosen invariance?
```
