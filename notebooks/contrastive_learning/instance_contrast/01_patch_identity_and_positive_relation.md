# Patch Identity And The Positive Relation

Instance contrast begins by assigning an identity to each sampled patch. That
identity is an algorithmic object, not a biological class.

Primary anchors:

- SimCLR: https://arxiv.org/abs/2002.05709
- DSMIL: https://arxiv.org/abs/2011.08939
- CTransPath: https://pubmed.ncbi.nlm.nih.gov/35952419/

## Hierarchical Data And Flattened Identities

Let slide `i` contain tissue patches:

```math
\mathcal{B}_i
=
\left\{
x_{ij}
\right\}_{j=1}^{n_i}.
```

The physical data-generating hierarchy is:

```math
I
\longrightarrow
X_{I,J}
\longrightarrow
V,
```

where `I` is a slide index, `J` is a patch index within the slide,
and `V` is an augmented view.

Instance pretraining flattens the hierarchy:

```math
N
=
\left(
I,J
\right)
\in
\left\{
1,\ldots,N_{\mathrm{patch}}
\right\}.
```

The identity variable is therefore a patch address, not:

```math
\text{diagnosis},
\qquad
\text{tissue type},
\qquad
\text{patient},
\qquad
\text{slide}.
```

## Positive Joint Distribution

Let `\mathcal{A}_1` and `\mathcal{A}_2` be stochastic view
kernels. A positive pair is generated by:

```math
N
\sim
p_N,
\qquad
V^{(1)}
\sim
\mathcal{A}_1
\left(
\cdot
\mid
x_N
\right),
\qquad
V^{(2)}
\sim
\mathcal{A}_2
\left(
\cdot
\mid
x_N
\right).
```

Thus:

```math
p_{+}
\left(
v_1,v_2
\right)
=
\sum_n
p_N(n)
\mathcal{A}_1
\left(
v_1
\mid
x_n
\right)
\mathcal{A}_2
\left(
v_2
\mid
x_n
\right).
```

This is a same-patch joint. It is not a same-morphology joint unless the
augmentation and sampling design make it one.

## Candidate Proposal

For an anchor view `A`, conventional instance discrimination draws one
positive key from the same identity and negatives from other sampled
identities:

```math
K^{+}
\sim
p
\left(
k
\mid
A
\right),
\qquad
K_r^{-}
\sim
\nu_K.
```

In an idealized full-dataset experiment:

```math
\nu_K
\left(
k
\right)
\approx
\sum_n
p_N(n)
\mathcal{A}_2
\left(
k
\mid
x_n
\right).
```

In a minibatch, `\nu_K` is instead induced by the patch sampler, slide
sampler, organ mixture, institution mixture, and augmentation pipeline.

## Population Critic

For the candidate-identification experiment, an unrestricted optimal critic
has the form:

```math
s^{\star}
\left(
a,k
\right)
=
\log
\frac{
p
\left(
k
\mid
a
\right)
}{
\nu_K(k)
}
+
c(a).
```

Therefore the identified relation is a density ratio between:

```math
\text{same-patch view conditional}
```

and:

```math
\text{sampler-induced key proposal}.
```

The score is not identified as a posterior probability of shared tissue type.

## Biological Latent State

Let `U_n` denote latent morphology and `D_n` denote acquisition
nuisance:

```math
x_n
\sim
p
\left(
x
\mid
U_n,D_n
\right).
```

The positive mechanism guarantees shared patch identity:

```math
N^{(1)}
=
N^{(2)}.
```

It does not separately guarantee that every visible biological component is
preserved:

```math
U
\left(
V^{(1)}
\right)
=
U
\left(
V^{(2)}
\right).
```

A crop can omit a rare focus, a color transform can erase a diagnostic stain
cue, and blur can remove nuclear detail. Those are errors in the declared
positive relation.

## Same Morphology Can Be A Negative

Suppose two distinct patches contain the same latent tissue state:

```math
N_a
\ne
N_b,
\qquad
U_a
=
U_b.
```

Conventional instance contrast places them in the denominator. The probability
of this event is:

```math
\Pr
\left(
U_a=U_b
\mid
N_a\ne N_b
\right),
```

which can be large in histology because normal epithelium, stroma, lymphocytes,
necrosis, and common tumor patterns recur within and across slides.

## Same Slide Does Not Imply A Positive

For two patches from the same slide:

```math
I_a
=
I_b,
\qquad
J_a
\ne
J_b,
```

the basic instance objective still treats the identities as different. Slide
membership is not a positive edge unless it is explicitly added.

Consequently, the positive graph contains edges between views of each patch:

```math
E_{+}^{\mathrm{instance}}
=
\left\{
\left(
t_1(x_n),t_2(x_n)
\right)
\right\}_{n=1}^{N_{\mathrm{patch}}},
```

but not all within-slide edges:

```math
\left(
x_{ij},x_{ik}
\right),
\qquad
j\ne k.
```

## C/R/G/S Placement

```text
\mathcal{G}:
    patch identities, stochastic views, and a sampler-induced candidate set

\mathcal{C}:
    shared or online/target patch encoders

\mathcal{R}:
    projected and usually normalized patch embedding

\mathcal{S}:
    identify the same-patch key among other patch identities
```

## Surviving Object

The pretraining statistic that survives is an embedding adapted to:

```math
\frac{
p
\left(
\text{view}
\mid
\text{same patch}
\right)
}{
p
\left(
\text{view under the candidate sampler}
\right)
}.
```

Any interpretation as morphology, diagnosis, or slide similarity requires an
additional assumption and downstream validation.
