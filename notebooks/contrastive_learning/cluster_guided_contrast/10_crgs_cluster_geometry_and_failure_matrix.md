# Cluster Geometry, C/R/G/S, And Failure Matrix

DeepCluster, SwAV, and RetCCL all use clusters, but the cluster survives as a
different mathematical object in each method.

Primary anchors:

- DeepCluster: https://arxiv.org/abs/1807.05520
- SwAV: https://arxiv.org/abs/2006.09882
- RetCCL: https://pubmed.ncbi.nlm.nih.gov/36270093/

## Unified Assignment Notation

Let:

```math
Z
=
\left\{
z_n
\right\}_{n=1}^{N}
```

be an embedding collection and:

```math
C
=
\left[
c_1,
\ldots,
c_K
\right]
```

be centroids or prototypes.

The generic assignment map is:

```math
\mathcal{A}
\left(
Z,C
\right)
\longrightarrow
Q.
```

The methods differ in the domain, codomain, and use of `Q`.

## DeepCluster Object

```math
Q_{\mathrm{DC}}
\in
\left\{
0,1
\right\}^{K\times N},
```

with:

```math
\mathbf{1}_K^{\top}
Q_{\mathrm{DC}}
=
\mathbf{1}_N^{\top}.
```

Each column is one-hot. The assignments are generated by full-dataset k-means
and then become categorical pseudo-labels.

### C/R/G/S

```text
\mathcal{G}:
    full-dataset Voronoi partition under preprocessed feature distance

\mathcal{C}:
    global k-means alternating with encoder and classifier updates

\mathcal{R}:
    hard pseudo-label; centroid discarded during prediction

\mathcal{S}:
    balanced cross-entropy prediction of assignments under augmentation
```

### Surviving Statistic

```math
\text{predictability of a repeatedly recomputed hard partition}.
```

## SwAV Object

For a minibatch:

```math
Q_{\mathrm{SwAV}}
\in
\mathbb{R}_{+}^{K\times B}
```

solves:

```math
\max_{
Q\in\mathcal{Q}
}
\mathrm{Tr}
\left(
Q^{\top}C^{\top}Z
\right)
+
\varepsilon H(Q).
```

Its marginals are balanced, and normalized columns become soft code targets.

### C/R/G/S

```text
\mathcal{G}:
    minibatch prototype-feature bipartite graph with uniform marginals

\mathcal{C}:
    shared encoder, trainable prototypes, and online Sinkhorn assignment

\mathcal{R}:
    soft balanced code

\mathcal{S}:
    swapped cross-view code prediction
```

### Surviving Statistic

```math
\text{view-consistent probability coordinates over learned prototypes}.
```

## RetCCL Objects

RetCCL has at least three cluster-derived objects:

```math
\text{memory subqueue membership},
```

```math
\text{cross-branch group centroid target},
```

and:

```math
\text{within-slide feature and spatial mosaic selection}.
```

### C/R/G/S For Pretraining

```text
\mathcal{G}:
    instance pairs, k-means memory subqueues, and cross-branch centroid sets

\mathcal{C}:
    shared encoder with g_1 group and g_2 instance heads

\mathcal{R}:
    piecewise weighted negative logits plus selected opposite-branch centroid

\mathcal{S}:
    weighted InfoNCE plus lambda times group-level InfoNCE
```

### C/R/G/S For Retrieval

```text
\mathcal{G}:
    feature-spatial mosaic database and query-to-archive patch edges

\mathcal{C}:
    dual clustering, cosine search, entropy ranking, filtering, and voting

\mathcal{R}:
    mosaic patch set, top-five retrieval bags, and source-slide votes

\mathcal{S}:
    archive diagnosis metadata enters nonparametric ranking
```

### Surviving Statistic

```math
\text{cluster-adjusted patch geometry plus a sparse slide-to-archive retrieval relation}.
```

## Method Matrix

| Axis | DeepCluster | SwAV | RetCCL |
|---|---|---|---|
| assignment scope | full dataset | minibatch plus optional feature queue | memory bank, minibatch branches, and individual WSI |
| assignment type | hard one-hot | soft balanced transport | hard k-means membership and selected centroids |
| centroid persistence | recomputed, then unused in CE | trainable prototypes | recomputed subqueue/group/mosaic centroids |
| balance mechanism | uniform pseudo-label sampling | exact soft marginal constraint | cluster-guided weighting and group discrimination |
| cross-view mechanism | augmented view predicts fixed pseudo-label | swapped code prediction | paired weighted InfoNCE and cross-branch group prediction |
| slide representation | none | none | selected feature-spatial mosaic |
| downstream readout | retained encoder | retained encoder | patch retrieval, entropy filter, top-five vote |

## Hard Versus Soft Geometry

DeepCluster maps:

```math
z
\longrightarrow
e_{\kappa(z)}.
```

The assignment is discontinuous at Voronoi boundaries.

SwAV maps the entire batch:

```math
Z_{\mathcal{B}}
\longrightarrow
Q^{\star}
\left(
Z_{\mathcal{B}},C
\right).
```

A sample's code depends on other batch samples through marginal constraints.

RetCCL maps:

```math
\left(
z,n
\right)
\longrightarrow
\phi(n)
```

for instance negatives and:

```math
z
\longrightarrow
\arg\max_r
z^{\top}S_r
```

for group targets. Both are hard cluster-conditioned switches.

## Balance Is A Prior

DeepCluster's balanced risk uses:

```math
\widetilde\pi_k
=
\frac{1}{K}.
```

SwAV imposes:

```math
\frac{1}{B}
\sum_{b=1}^{B}
q_b
=
\frac{1}{K}\mathbf{1}_K.
```

Both prevent dominant-group collapse by changing the effective assignment
prior. Neither equality estimates pathology tissue prevalence.

RetCCL instead modifies negatives from one key-nearest subqueue:

```math
\phi(n)
\in
\left\{
w,1
\right\}.
```

This is a local cluster-conditioned correction, not global equipartition.

## Gradient Objects

### DeepCluster

For fixed pseudo-label `y_n`:

```math
\nabla_{z_n}
\ell_{\mathrm{CE}}
=
W
\left(
p_n-y_n
\right).
```

The centroid does not appear in the network-update gradient.

### SwAV

```math
\nabla_z
\ell
\left(
z,q
\right)
=
\frac{1}{\tau}
C
\left(
p-q
\right).
```

The gradient is a difference of predicted and target prototype barycenters.

### RetCCL Weighted Instance Branch

```math
\frac{
\partial\ell
}{
\partial s_i^{-}
}
=
\frac{
\phi_i
}{
\tau
}
\pi_i.
```

Cluster membership scales the negative similarity gradient.

### RetCCL Group Branch

```math
\nabla_g
\ell_{\mathrm{group}}
=
\frac{1}{\tau}
\left(
\sum_r
\pi_r S_r
-
S_{r^{\star}}
\right).
```

The gradient predicts the currently nearest opposite-branch centroid.

## Failure Matrix

| Failure | DeepCluster | SwAV | RetCCL |
|---|---|---|---|
| single-cluster collapse | empty-cluster repair and balanced sampler | balanced transport forbids single prototype use | multiple losses and subqueues, but no universal guarantee |
| incorrect balance prior | inverse-size sampling | uniform prototype marginals | cluster weighting can misidentify false negatives |
| assignment discontinuity | hard nearest centroid | soft codes reduce it; low entropy can sharpen it | hard nearest subqueue and nearest group centroid |
| temporal inconsistency | epochwise reassignment | online prototypes and optional historical queue | memory-bank and online cluster drift |
| artifact cluster amplification | rare clusters overweighted | every prototype receives mass | artifact subqueue can alter many negative weights |
| false semantic grouping | pseudo-label predicted as class | code consistency reinforces prototype | nearest group centroid and subqueue reinforced |
| no slide layout | patch encoder only | patch encoder only | spatial sampling without relational layout metric |
| rare patch removal | not a slide method | not a slide method | mosaic and top-five filters can discard it |

## A Slide Counterexample

Let two slides contain the same patch-feature multiset:

```math
\left\{
z_{aj}
\right\}_{j=1}^{n}
=
\left\{
z_{bj}
\right\}_{j=1}^{n},
```

but different spatial arrangements:

```math
\mathcal{R}_a
\ne
\mathcal{R}_b.
```

DeepCluster and SwAV pretraining cannot distinguish slide arrangement because
they operate on patches.

RetCCL's spatial clustering can select representatives from different regions,
but if the final mosaics retain the same feature set and discard relational
coordinates:

```math
\mathcal{M}_a
=
\mathcal{M}_b,
```

feature-cosine retrieval also collides. Spatially diverse sampling is weaker
than preserving a graph or coordinate relation.

## Complexity Matrix

Let `N` be dataset size, `B` batch size, `K` cluster count,
`d` feature dimension, `I` assignment iterations, and `M`
archive mosaic size.

| Method | Dominant cluster/context cost |
|---|---|
| DeepCluster | full feature pass plus approximately `\Theta(INKd)` k-means distance work |
| SwAV | `\Theta(BKd+IBK)` prototype scoring and Sinkhorn scaling per assignment batch |
| RetCCL pretraining | memory clustering plus contrastive and centroid candidate comparisons |
| RetCCL retrieval | offline patch encoding and clustering; naive online `\Theta(kMd)` patch search |

## Design Axes

The family exposes independent choices:

| Axis | Alternatives |
|---|---|
| assignment support | full cohort, minibatch, memory, slide, region |
| assignment hardness | one-hot, entropy-regularized soft, posterior-weighted |
| balance | none, inverse size, uniform transport, known tissue prior |
| temporal update | epochwise, online, EMA memory, frozen prototypes |
| supervision use | pseudo-label, swapped code, logit weight, centroid target, retrieval selector |
| slide readout | none, pooled code histogram, graph, mosaic, archive query |

## Bottom Line

The word "cluster" does not specify the method. The mathematically relevant
question is:

```math
\text{Which assignment is computed, over what support, under which prior, and
where does it enter the gradient or readout?}
```

DeepCluster preserves hard partition predictability, SwAV preserves balanced
cross-view prototype coordinates, and RetCCL uses clusters to reshape both
patch learning and slide retrieval.
