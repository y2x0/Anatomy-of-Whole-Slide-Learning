# Self-Training As Latent-Variable Optimization

Pseudo-labeling can be written as approximate latent-variable optimization.

Let $Z_i$ be hidden instance labels and $Y_i$ observed bag labels.

The ideal objective is:

```math
\max_\theta
\sum_i
\log
P_\theta(Y_i\mid H_i)
=
\max_\theta
\sum_i
\log
\sum_{z_i}
P_\theta(Y_i,z_i\mid H_i).
```

The sum over $z_i$ is often hard or poorly identified. Pseudo-labeling replaces
the posterior with a model-generated point estimate:

```math
\widehat z_{i,t}
=
\Psi_t(H_i,Y_i,\theta_t,\mathcal{D}).
```

Then it optimizes:

```math
\max_\theta
\sum_i
\log
P_\theta(Y_i,\widehat z_{i,t}\mid H_i).
```

This is hard EM in spirit only after a probabilistic model is specified. If the
paper only defines a heuristic selection rule, the correct statement is weaker:
the method behaves like a hard-assignment training procedure, but it is not an
EM algorithm for a stated likelihood.

## EM View

The exact evidence lower bound is:

```math
\log P_\theta(Y_i\mid H_i)
\ge
\mathbb{E}_{q_i(z)}
\left[
\log P_\theta(Y_i,z\mid H_i)
-
\log q_i(z)
\right].
```

The E-step would choose:

```math
q_i(z)
=
P_{\theta^{old}}(z\mid Y_i,H_i).
```

Hard pseudo-labeling chooses:

```math
q_i(z)
=
\delta_{\widehat z_{i,t}}(z).
```

The M-step then treats $\widehat z_{i,t}$ as truth. This is legitimate EM only
when:

```math
\widehat z_{i,t}
=
\arg\max_z
P_{\theta_t}(z\mid Y_i,H_i)
```

under the same likelihood later optimized in the M-step. Attention top-k,
confidence thresholding, or teacher targets usually violate this exact
condition.

## Confidence Thresholding

A pseudo-label rule may use:

```math
\widehat z_{ij}
=
\mathbf{1}\{p_\theta(Z_{ij}=1\mid h_{ij})\ge\tau\}.
```

Only confident predictions are used:

```math
M_{ij}
=
\mathbf{1}
\left\{
\max_c p_\theta(Z_{ij}=c\mid h_{ij})
\ge
\tau
\right\}.
```

The pseudo-label loss is:

```math
\mathcal{L}_{\mathrm{pseudo}}
=
-
\sum_{i,j}
M_{ij}
\log
p_\theta(\widehat z_{ij}\mid h_{ij}).
```

The mask is not missing completely at random. It is generated by the model:

```math
P(M_{ij}=1\mid Z_{ij},H_i,G_i,\theta_t)
\ne
P(M_{ij}=1).
```

More explicitly, thresholding selects examples as a function of confidence:

```math
M_{ij}
=
m_\tau(p_{\theta_t}(\cdot\mid h_{ij})).
```

That makes it a model-generated MNAR selection process. The supervised loss is
estimating risk under the selected distribution:

```math
P(H,Z\mid M=1),
```

not automatically under the target distribution:

```math
P(H,Z).
```

## Coupled Target Problem

Pseudo-labels depend on parameters:

```math
\widehat z
=
\Psi_{\theta}(H,Y,\mathcal{D}).
```

The training target changes as the model changes:

```math
\mathcal{L}(\theta)
=
\ell(\theta,\Psi_{\theta}(H)).
```

Most implementations stop gradients through $\Psi_\theta$:

```math
\nabla_\theta
\ell(\theta,\mathrm{stopgrad}(\Psi_{\theta}(H))).
```

This is not optimizing an ordinary fixed supervised loss.

## C/R/G/S Placement

```text
G:
    pseudo-labels may use spatial neighborhoods, clusters, or graph context

C:
    pseudo-instance labels directly shape feature space

R:
    pseudo-label rules often select which instances influence readout

S:
    observed weak labels plus generated latent assignments
```

## Dense Summary

Pseudo-labeling replaces posterior uncertainty:

```math
P(Z\mid Y,H)
```

with a point estimate:

```math
\widehat Z_t.
```

It gains stronger gradients by pretending an inferred latent variable is
observed. The danger is that the inferred variable may be wrong and that the
set of examples receiving pseudo-labels is itself a biased, model-dependent
sample.
